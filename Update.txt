v1.2-alpha (Wiederdämmerung)
相对于 v1.1-alpha 的更新内容如下：

本次更新基于一次更完整的工程级代码审查与后续功能补强，重点修复了多处影响正确性、稳定性、跨后端一致性与用户体验的关键问题，并同步加入若干 GUI、抓包、配置导入导出与底层性能优化改进。

[新增]
1. 新增底部状态栏实时进度反馈
   - BottomBar 不再仅显示静态百分比
   - 运行过程中可实时显示：
     进度百分比 / 已耗时 / 预计剩余时间（ETA）
   - 完成后会显示总耗时
   - 进度显示改为更适合长时间运算观察的双行结构，并使用等宽字体减少数字跳动感

2. 新增配置码压缩格式
   - 配置导出不再仅使用 base64(json)
   - 新增压缩配置码格式：
     Z1: + base64(zstd(json_compact))
   - 配置码长度显著缩短，便于复制、分享与保存
   - 同时保持对旧版纯 base64 配置码的兼容，可直接导入旧配置

3. 新增强适配的基准测试规模控制
   - 基准测试不再使用固定模组数量
   - 现在会根据 combo_size 动态计算更合适的 N，使组合规模落在较合理区间
   - 避免低件数测试过快失真，也避免高件数测试直接卡死
   - 提高不同组合件数下的可比性与可用性

[优化]
1. 优化抓包依赖体积与冷启动速度
   - packet_capture.py 改为从 scapy_minimal 导入 sniff / IP / TCP / Raw
   - 避免加载大量未使用协议层模块
   - 有助于减小打包体积、缩短启动时间并降低运行时内存占用

2. 优化配置导入导出体验
   - 导出时使用更紧凑的 JSON 序列化方式
   - 导入时自动识别新旧两种格式
   - 无需用户手动区分版本

3. 优化基准测试可用性
   - 不同 combo_size 下自动匹配更合理的测试规模
   - 避免原先固定规模导致的测试结果失真与界面卡顿

4. 优化 C++ / CUDA 组合索引解包性能
   - 组合定位逻辑由线性扫描改为基于组合数单调性的二分搜索
   - 在大规模组合场景下显著降低起始组合索引恢复开销
   - 有助于 CPU 与 CUDA 路径整体初始化性能提升

5. 优化 C++ 大规模枚举时的内存峰值
   - 调整局部 Top-K 裁剪前的额外缓冲策略
   - 减少解向量在裁剪前过度膨胀的问题
   - 提高大规模枚举时的内存稳定性

6. 优化 CUDA 资源清理结构
   - 将重复的显存释放逻辑提取复用
   - 提高代码可维护性并降低后续修改出错概率

[修复]
1. 修复排除属性筛选完全不生效的问题（严重）
   - exclude_attributes 参数虽然可从 CLI / GUI 传入
   - 但旧版筛选逻辑中从未真正执行排除判断
   - 现已补上排除属性过滤，包含排除属性的模组会被正确跳过

2. 修复 Python 侧 TOTAL_ATTR_POWER_MAP 大面积缺失导致战斗力计算偏低的问题（严重）
   - 补全缺失的总属性值到战斗力映射
   - 使 Python 侧与 C++ 侧映射保持一致
   - 避免部分总属性值落入缺失区间时被错误映射为 0

3. 修复 _restore_original_scores 未对总属性值执行 120 上限裁切的问题（严重）
   - Python 侧现已加入 min(total_attr_value, 120)
   - 与 C++ 路径保持一致
   - 避免总属性值超过上限时查表失败并回落为 0

4. 修复 mod_info 为 None 时可能触发 AttributeError 崩溃的问题（严重）
   - 在解析模组详情前加入空值检查
   - 缺失详情时跳过对应项并记录调试信息
   - 防止解析流程直接崩溃

5. 修复 enumerate_modules 与 optimize_modules 评分还原触发条件不一致的问题
   - 旧版仅部分路径会触发 _restore_original_scores
   - 现在统一判断逻辑
   - 避免仅设置 min_attr_sum_requirements 时枚举结果仍显示 boost 后分数

6. 修复 os._exit(0) 导致的不安全进程退出问题
   - 改为 sys.exit(0)
   - 保证 finally、资源释放、日志写入与退出流程可正常执行

7. 修复优化失败时异常被静默吞掉的问题
   - 旧版 except Exception 仅记录日志后 return
   - 导致上层无法感知优化失败，用户可能一直等待无结果
   - 现已在记录后重新抛出异常，并保证 SystemExit 正常传播

8. 修复 star_railway_monitor.py 中 SystemExit 被误拦截的问题
   - 防止正常退出流程被 except Exception 错误吞掉
   - 避免程序在优化完成后陷入无意义等待

9. 修复自动检测网络接口匹配错误的问题
   - 修正对 route print 输出列的解析
   - 由错误匹配 Gateway 改为正确匹配 Interface
   - 提高网卡自动识别的正确性

10. 修复 module_parser.py 中 break 导致同 package 后续模组丢失的问题
    - 当某个 item 缺少 ModNewAttr 时
    - 旧版会 break 提前终止整个包的后续遍历
    - 现已改为 continue，确保后续 item 不会被错误跳过

11. 修复 OpenCL 无效解哨兵值与 CUDA 不一致的问题
    - 旧版 OpenCL 将无效解记为 score = 0
    - 可能污染 Top-K 结果
    - 现已改为负值哨兵，与 host 端统一过滤逻辑对齐

12. 修复 optimize_modules 在非 4 件组合时退回纯枚举的问题
    - 删除 combo_size != 4 的特殊早返回
    - 所有 1~10 件组合统一走完整优化流程

13. 修复 C++ 核心索引打包 / 解包对 4 件组合的硬编码问题
    - 修正 CompactSolution 多件数组合下的索引恢复逻辑
    - 同时避免低件数时误读填充数据

14. 修复 OpenCL 后端对 combo_size 支持不完整的问题
    - 补齐实际件数传递与相关组合处理逻辑
    - 使非 4 件组合不再被迫回退

15. 修复 CUDA 后端多处稳定性与正确性问题
    - 修复设备端数组初始化不稳定
    - 修复阈值评分循环未提前终止
    - 修复无效解与合法零分解混淆
    - 修复索引槽位边界保护不足
    - 修复显存估算遗漏 compact 缓冲区的问题
    - 统一属性值上限裁切写法

16. 修复 CUDA 结果复核与多件数组合索引恢复问题
    - 增加 GPU 结果回传后的 CPU 精确复核
    - 修复 4 件以上组合在 CUDA 路径下的索引恢复与排序正确性

17. 修复抓包与调试模块中的兼容性问题
    - 修复 notify_dumper 对空 bytes 结果的判断问题
    - 改进 packet_capture 的兼容层、事件输出与主流程适配能力

18. 修复 CUDA 阈值选择路径缺少错误检查的问题
    - 为 cudaMalloc / cudaMemcpy / kernel launch / 同步等关键步骤加入返回值检查
    - 防止异常情况下阈值错误回落并污染后续 compact 过程

19. 修复 CUDA 属性聚合槽位硬编码散落的问题
    - 将多个 32 槽位魔法数字提取为统一常量
    - 降低未来属性种类扩展时的修改风险

20. 修复 CPU 批次上限使用魔法数字的问题
    - 将未经说明的硬编码数值提取为具名常量
    - 补充注释说明来源与用途

4. 调整底部状态栏布局与信息密度
