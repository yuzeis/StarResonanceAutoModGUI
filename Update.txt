StarResonanceAutoModGUI
Update.log

v1.1
相对于 v1.0 的更新内容如下：

[新增]
1. 新增配置码导入 / 导出功能
   - 支持将当前 GUI 配置序列化为配置码并导出
   - 支持通过配置码一键导入并恢复筛选条件、属性选择、件数、调试选项等设置
   - 便于跨设备、跨环境快速复用配置

2. 新增抓包与调试辅助组件
   - 新增 scapy_minimal.py
   - 新增 notify_dumper.py
   - 新增 / 更新 packet_capture.py
   - 上述文件为从 StarResonanceTrade v5.5-rc 相关实现思路与结构移植、整理并适配本 GUI 工程而来
   - 用于优化抓包、协议解析、最小化依赖与调试落盘流程

3. 新增模组数量动态反馈
   - 扫描过程中可自动解析日志中的模组总数
   - 主界面可基于实际扫描结果动态估算组合数 C(N, k)
   - 当尚未完成扫描时，预估区域会明确提示需先运行一次扫描

[优化]
1. 优化基准测试规模
   - 基准测试默认构造模组数量由 500 调整为 200
   - 降低无意义超大规模压测带来的耗时与卡顿
   - 提高 GUI 基准测试可用性与响应速度

2. 优化配置恢复能力
   - 属性选择器支持直接恢复选中项
   - 最小属性和约束条件支持直接回填
   - 提高配置导入后的状态一致性

3. 优化组合数与时间预估逻辑
   - 不再使用过于粗略的固定推测值
   - 改为优先基于真实扫描结果估算
   - 当数据不足时提供更明确的提示信息

4. 优化抓包架构
   - PacketCapture 重构为更清晰的事件驱动与缓冲处理结构
   - 改进 TCP 重组、缓存管理、停止控制与统计信息输出
   - 增强整体稳定性与后续扩展性

[修复]
1. 修复 optimize_modules 在非 4 件组合时退回纯枚举的问题
   - 删除 combo_size != 4 的特殊早返回
   - 统一走完整优化流程，与 1~10 件组合支持保持一致

2. 修复 C++ 核心索引打包 / 解包对 4 件组合的硬编码问题
   - 完善 CompactSolution 解包逻辑
   - 修复多件数组合时索引恢复不完整、低件数时误读填充数据的问题

3. 修复 OpenCL 后端对 combo_size 支持不完整的问题
   - 补充实际件数参数传递
   - 修复总组合数计算、Kernel 内部组合处理、结果打包与解包逻辑
   - 移除非 4 件组合时的不必要回退限制

4. 修复 CUDA 后端多处稳定性与正确性问题
   - 修复设备端数组初始化不稳定
   - 修复阈值评分循环未提前终止
   - 修复无效解与合法零分解混淆
   - 修复索引槽位边界保护不足
   - 修复显存估算遗漏 compact 缓冲区的问题
   - 统一属性值上限裁切写法

5. 修复 CUDA 结果复核与多件数组合索引恢复问题
   - 增加 GPU 结果回传后的 CPU 精确复核
   - 修复 4 件以上组合在 CUDA 路径下的索引恢复与排序正确性

6. 修复抓包与调试模块中的兼容性问题
   - 修复 notify_dumper 对空 bytes 结果的判断问题
   - 改进 packet_capture 的兼容层、事件输出与主流程适配能力

[调整]
1. 调整 OpenCL 路径对非 4 件组合的限制
   - 不再沿用旧版 combo_size != 4 时直接回退 CPU 的限制思路
   - 在后端完整支持后，统一按实际组合件数处理

2. 更新部分底层注释与接口说明
   - 调整 module_optimizer 相关头文件注释
   - 补充多件数组合兼容说明与索引打包语义
   - 提升可维护性与后续扩展性

[兼容性说明]
1. 本次更新重点集中在：
   - 组合件数不固定时的正确性修复
   - CUDA / OpenCL 后端结果一致性修复
   - GUI 配置体验增强
   - 抓包模块稳定性与调试能力改进

2. v1.1 对 1~10 件组合的支持链路进行了补强
   - 包括 Python 调度层
   - C++ 核心索引打包 / 解包
   - OpenCL 计算与结果回传
   - CUDA 枚举、筛选、紧凑输出与复核流程

3. 新增的 scapy_minimal.py、notify_dumper.py、packet_capture.py
   - 为从 StarResonanceTrade v5.5-rc 移植并适配的组件
   - 已按本工程结构与 GUI 使用场景进行裁剪与整合
